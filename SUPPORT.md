# Support

Thanks for using **fmtly**.

## Where to ask for help

- **Bug reports:** Open a GitHub issue using the **Bug report** template.
- **Feature ideas:** Open a GitHub issue using **Feature request** or **Tool proposal**.
- **Documentation fixes:** Open a GitHub issue using **Documentation improvement**.

## Security issues

For security vulnerabilities, **do not** open a public issue.

- Email: **oguzhankir17@gmail.com**
- Policy: [SECURITY.md](SECURITY.md)

## Before opening an issue

- Read [CONTRIBUTING.md](CONTRIBUTING.md)
- Check existing issues for duplicates
- Reproduce on the latest `main`
- Include minimal reproduction input/output (without private data)

## Response expectations

As an open-source project, response times can vary.

- Critical bugs and security reports are prioritized.
- Well-structured reports with clear repro steps are handled faster.

Thanks for contributing and helping improve fmtly.
